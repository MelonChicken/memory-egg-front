## Git Workflow / Branch Strategy

This project uses a simple branch-based workflow.

### Main Branches

- `main`: Stable branch for final submission and evaluation.
- `dev`: Integration branch where completed features are merged before release.
- `feature/*`: Feature branches used by each member for specific tasks.

### Feature Branch Naming

Use descriptive feature branch names:

```text
feature/auth
feature/egg-dashboard
feature/posts
feature/quests
feature/shop-inventory
feature/frontend-layout
feature/ai-verification
